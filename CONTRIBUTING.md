# Contributing to ir-loader-g

Thank you for your interest in contributing! To keep things minimalist and efficient, please follow these simple guidelines:

## Pull Requests
1. Fork the repository and create your feature branch from `main`.
2. Ensure your code compiles correctly without warnings via CMake.
3. Keep the codebase minimal; avoid introducing heavy external dependencies unless absolutely necessary.
4. Do not commit heavy binaries (`.lib`, `.dll`, `.exe`, `.vst3`, etc.). Ensure your `.gitignore` is active.
5. Open a Pull Request with a clear, concise description of your changes.

## Reporting Issues
- Use the GitHub Issues tab.
- Describe the bug or feature request clearly.
- If it's a bug, include your OS, DAW, and steps to reproduce.
